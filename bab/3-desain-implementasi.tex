Diagram \ref{fig:arsitektursistem} menggambarkan arsitektur teknis dari sistem
marketplace tiket konser berbasis NFT yang dibangun menggunakan teknologi
blockchain, khususnya pada jaringan Polygon zkEVM. Arsitektur ini dibagi
menjadi empat komponen utama: Frontend, Smart Contract, Ethereum Virtual
Machine (EVM), dan Polygon zkEVM, yang masing-masing memiliki peran penting
dalam membentuk pengalaman pengguna dan pengelolaan aset digital berbasis
Non-Fungible Token (NFT).

Komponen pertama adalah Frontend, yang merupakan antarmuka pengguna dari
aplikasi web. Di dalamnya digunakan framework Next.js, sebuah kerangka kerja
React yang mendukung server-side rendering, static site generation, dan dynamic
routing, yang berguna untuk menampilkan halaman dinamis seperti daftar event,
tiket yang dimiliki pengguna, serta proses pembelian. Frontend juga menggunakan
ethers.js, sebuah pustaka JavaScript yang dirancang untuk berinteraksi dengan
blockchain Ethereum dan kompatibelnya (seperti Polygon zkEVM). Ethers.js
memungkinkan aplikasi frontend untuk melakukan transaksi blockchain, memanggil
fungsi-fungsi smart contract, serta mengelola koneksi dengan wallet pengguna
seperti MetaMask.

Selanjutnya, bagian Smart Contract berperan sebagai tulang punggung logika
bisnis aplikasi di sisi blockchain. Smart contract ini ditulis menggunakan
bahasa pemrograman Solidity, yang merupakan bahasa standar untuk membuat
kontrak cerdas di ekosistem Ethereum. Dalam konteks marketplace tiket, smart
contract bertanggung jawab untuk menangani pembuatan event, pencetakan tiket
dalam bentuk NFT, pengelolaan status tiket (aktif atau telah digunakan), serta
distribusi hasil penjualan tiket termasuk pemotongan platform fee. Smart
contract ini juga mengatur hak akses khusus seperti organizer, verifier, dan
pemilik tiket. Tiket konser dalam sistem ini direpresentasikan sebagai NFT
(Non-Fungible Token). NFT adalah token unik yang tidak dapat dipertukarkan satu
sama lain, sehingga cocok digunakan untuk mewakili aset digital yang memiliki
identitas individual seperti tiket. Setiap tiket memiliki informasi spesifik
seperti ID acara, pemilik, waktu pembelian, status penggunaan, dan metadata
gambar khusus. Token NFT ini dicetak langsung melalui kontrak pintar dan
disimpan di dompet pengguna setelah pembelian berhasil dilakukan.

Komponen ketiga adalah Ethereum Virtual Machine (EVM), yang merupakan
lingkungan eksekusi standar untuk smart contract berbasis Ethereum. EVM
bertindak sebagai mesin yang menjalankan kode Solidity yang telah dikompilasi
menjadi bytecode. Keunggulan menggunakan EVM adalah interoperabilitasnya dengan
berbagai jaringan layer-2 yang mendukung EVM, sehingga kontrak yang ditulis
untuk Ethereum juga dapat berjalan di jaringan seperti Polygon zkEVM tanpa
perubahan kode signifikan. Komponen terakhir adalah Polygon zkEVM, sebuah
solusi layer-2 scaling yang dibangun dengan teknologi zero-knowledge rollup
(zk-Rollup) dan kompatibel dengan EVM. Polygon zkEVM dirancang untuk memberikan
performa tinggi dengan biaya transaksi yang rendah serta waktu konfirmasi yang
cepat, tanpa mengorbankan keamanan karena datanya tetap dapat diverifikasi di
jaringan Ethereum utama. Dengan menggunakan zkEVM, sistem marketplace dapat
memanfaatkan keunggulan skalabilitas dan efisiensi biaya, sambil tetap
mempertahankan fungsionalitas dan keamanan dari ekosistem Ethereum.

Interaksi antara komponen-komponen ini memungkinkan pengguna untuk mengakses
platform melalui web, menghubungkan dompet kripto mereka menggunakan MetaMask,
menjelajahi acara konser, membeli tiket NFT secara langsung dari smart
contract, serta menggunakan tiket tersebut dengan mekanisme verifikasi
on-chain. Semua transaksi yang dilakukan akan dicatat secara permanen di
blockchain Polygon zkEVM, memberikan transparansi, imutabilitas, dan verifikasi
yang tidak dapat dimanipulasi. Dengan arsitektur ini, sistem tidak hanya
mengotomatisasi penjualan tiket dan penggunaan tiket, tetapi juga menciptakan
model distribusi digital yang adil, aman, dan tahan terhadap pemalsuan.

Diagram alur di atas menggambarkan proses pembelian tiket konser berbasis NFT dalam sistem marketplace yang dibangun menggunakan teknologi blockchain. Proses dimulai ketika pengguna mengakses halaman event pada antarmuka aplikasi web. Halaman ini menyajikan berbagai informasi terkait suatu acara, seperti nama konser, lokasi, waktu pelaksanaan, daftar artis, harga tiket, dan jumlah tiket yang tersedia. Di halaman ini, pengguna akan disajikan tombol atau antarmuka yang memungkinkan mereka untuk memulai proses pembelian tiket secara langsung.

Ketika pengguna menekan tombol untuk membeli tiket, sistem akan terlebih dahulu
memeriksa apakah wallet kripto pengguna sudah terhubung ke aplikasi atau belum.
Wallet ini merupakan elemen penting dalam ekosistem Web3 karena berfungsi
sebagai identitas digital pengguna sekaligus tempat penyimpanan aset kripto dan
NFT. Wallet seperti MetaMask harus terhubung agar pengguna dapat berinteraksi
dengan smart contract yang mengatur penerbitan dan distribusi tiket.

Jika pengguna belum menghubungkan wallet-nya, sistem akan menampilkan
notifikasi atau prompt yang menginformasikan bahwa wallet perlu dihubungkan
terlebih dahulu untuk melanjutkan proses pembelian. Pengguna kemudian akan
diarahkan untuk menghubungkan wallet mereka melalui pop-up atau jendela
eksternal dari ekstensi wallet. Proses ini juga mencakup proses validasi
jaringan, di mana pengguna harus terhubung ke jaringan blockchain yang sesuai,
misalnya jaringan Polygon zkEVM jika sistem ini berjalan di atasnya. Setelah
wallet terhubung, sistem akan melanjutkan ke langkah berikutnya.

Jika wallet sudah terhubung, aplikasi akan menampilkan formulir pembelian
tiket, yang biasanya mencakup detail seperti jumlah tiket yang ingin dibeli,
nomor kursi (jika sistem mendukung penomoran tempat duduk), dan pratinjau dari
metadata NFT (misalnya gambar atau informasi yang melekat pada tiket digital).
Setelah pengguna mengisi formulir ini, mereka akan diarahkan ke tahap
konfirmasi pembelian, di mana sistem akan menampilkan ringkasan
transaksi—termasuk harga tiket, biaya platform (platform fee), dan estimasi gas
fee yang akan ditarik dari wallet.

Setelah konfirmasi dilakukan, proses dilanjutkan ke tahap pembayaran, di mana
transaksi akan dikirim ke jaringan blockchain melalui wallet pengguna. Smart
contract akan menerima pembayaran, mencetak NFT tiket, dan mengirimkan token
tersebut ke alamat wallet pengguna. Seluruh proses ini dicatat secara permanen
di blockchain, memastikan transparansi, keamanan, dan ketertelusuran aset
digital. Tiket yang dibeli kemudian dapat digunakan atau diverifikasi pada saat
hari acara berlangsung.

Keseluruhan alur ini menggambarkan bagaimana sistem Web3 mengintegrasikan aspek
tradisional seperti pembelian tiket dengan komponen teknologi baru seperti
wallet kripto dan smart contract. Dengan pendekatan ini, pembelian tiket
menjadi lebih transparan, tidak dapat dipalsukan, dan memberikan kendali penuh
kepada pengguna atas aset digital yang dimilikinya.
\subsection{Konfigurasi Environment}
Langkah pertama dalam pengembangan platform marketplace tiket konser berbasis
smart contract pada jaringan Layer-2 Ethereum adalah menyiapkan environment
pengembangan yang dapat mendukung pembuatan dan pengujian smart contract.
Proses ini dimulai dengan menginstal alat pengembangan blockchain seperti
Hardhat. Hardhat memberikan fleksibilitas lebih besar dalam hal penyesuaian dan
integrasi dengan jaringan Layer-2. Setelah memilih alat pengembangan yang
tepat, langkah berikutnya adalah mengonfigurasi jaringan testnet Polygon zkEVM
yang dapat diintegrasikan dengan Ethereum untuk memberikan transaksi yang lebih
cepat dan biaya yang lebih rendah. Selain itu, penting untuk menyiapkan dompet
kripto seperti MetaMask yang berfungsi untuk mengelola transaksi dan interaksi
dengan smart contract di jaringan Layer-2. MetaMask memungkinkan pengembang dan
pengguna untuk menghubungkan aplikasi dengan jaringan Ethereum atau Layer-2,
serta mengelola token dan NFT yang digunakan dalam platform. Pengguna juga
harus memverifikasi bahwa mereka memiliki akses ke Ethereum testnet atau
testnet Layer-2 yang relevan untuk menguji smart contract sebelum diterapkan
pada jaringan utama. Penggunaan testnet memungkinkan pengembang untuk
menghindari biaya transaksi yang tinggi dan risiko yang terkait dengan
pengujian pada jaringan utama. Untuk memulai pengembangan di jaringan Polygon
zkEVM, langkah pertama adalah menyiapkan Cardona testnet di MetaMask. Langkah
ini dimulai dengan membuka MetaMask dan memilih opsi untuk menambahkan jaringan
baru di pengaturan. Setelah itu, Anda perlu memasukkan rincian konfigurasi
Cardona testnet, seperti URL RPC, ID chain, nama jaringan, simbol mata uang
(ETH), dan URL Explorer.

\subsection{Pengembangan Smart Contract}
Smart contract dikembangkan menggunakan Solidity versi 0.8.28, bahasa
pemrograman utama untuk blockchain Ethereum dan Polygon zkEVM yang menyediakan
syntax yang familiar bagi developer dengan latar belakang JavaScript atau C++.
Solidity merupakan statically-typed programming language yang specifically
designed untuk mengembangkan smart contracts yang berjalan pada Ethereum
Virtual Machine (EVM), dengan features seperti inheritance, libraries, dan
complex user-defined types yang memungkinkan pengembangan aplikasi blockchain
yang sophisticated dan secure. Versi 0.8.28 yang digunakan dalam penelitian ini
memberikan built-in overflow/underflow protection, enhanced error handling
mechanisms, dan improved gas optimization features yang crucial untuk
cost-effective operations pada Layer-2 networks seperti Polygon zkEVM.Untuk
pengembangan, testing, dan deployment, Hardhat digunakan sebagai primary
development framework yang memungkinkan kompilasi, testing, deployment smart
contract, dan comprehensive debugging capabilities. Hardhat menyediakan
ekosistem lengkap yang terintegrasi dengan dukungan untuk advanced scripting
menggunakan JavaScript/TypeScript, automated testing menggunakan Mocha test
framework dan Chai assertion library, serta powerful debugging tools termasuk
console.log functionality dalam smart contracts, stack traces untuk failed
transactions, dan gas usage analysis. Framework ini juga menyediakan built-in
local blockchain environment melalui Hardhat Network yang memungkinkan rapid
development cycles dengan instant mining, configurable gas prices, dan detailed
transaksi logging untuk comprehensive testing scenarios.

Hardhat configuration memungkinkan multi-network deployment dengan easy
switching antara development, testnet, dan mainnet environments melalui single
configuration file yang mendefinisikan network parameters, account management,
dan deployment strategies. Plugin ecosystem yang extensive mencakup integrasi
dengan tools seperti Etherscan untuk contract verification, OpenZeppelin untuk
security auditing, gas reporting untuk cost analysis, dan coverage tools untuk
testing completeness measurement. Hardhat Tasks system memungkinkan creation
custom scripts untuk automated workflows seperti contract deployment, data
seeding, dan administrative operations yang dapat dijalankan dengan consistent
parameters across different environments. Development workflow menggunakan
Hardhat meliputi beberapa tahapan yang systematic: contract development dengan
real-time compilation checking, comprehensive unit testing menggunakan test
fixtures dan mock contracts, integration testing dengan simulated user
interactions, gas optimization analysis untuk identifying cost bottlenecks, dan
automated deployment scripts yang ensure consistent deployment across multiple
networks. Hardhat Console menyediakan interactive JavaScript environment yang
memungkinkan real-time interaction dengan deployed contracts, useful untuk
debugging, data querying, dan administrative operations tanpa need untuk
writing separate scripts. Testing framework yang robust mencakup unit tests
untuk individual contract functions, integration tests untuk multi-contract
interactions, stress tests untuk high-volume scenarios, dan edge case testing
untuk security vulnerabilities. Mocha test runner memberikan structured test
organization dengan describe blocks, before/after hooks untuk setup/teardown
operations, dan detailed test reporting dengan gas usage metrics. Chai
assertion library memungkinkan expressive test assertions dengan custom
matchers untuk blockchain-specific operations seperti event emission checking,
revert reason validation, dan state change verification. Plugin architecture
Hardhat memungkinkan extensibility melalui community-developed plugins yang
menambahkan functionality seperti automatic contract size checking untuk
deployment limits, security analysis integration dengan tools seperas Slither
atau MythX, dan performance monitoring untuk transaksi execution metrics.
Hardhat Verify plugin memungkinkan automatic source code verification pada
block explorers, penting untuk transparency dan user confidence dalam
production deployments. Gas reporter plugin memberikan detailed analysis biaya
operasi untuk setiap function call, memungkinkan optimization decisions
berdasarkan actual usage patterns. Configuration management melalui
hardhat.config.js file memungkinkan sophisticated setup untuk different
deployment scenarios, termasuk network-specific gas prices, account management
dengan hardware wallet integration, dan environment-specific contract
parameters. Compiler settings dapat dikonfigurasi untuk optimization levels
yang berbeda, target EVM versions untuk compatibility dengan different
networks, dan output selections untuk generating comprehensive compilation
artifacts. Network configuration mendukung multiple RPC providers dengan
automatic failover, custom gas estimation strategies, dan timeout
configurations untuk reliable network interactions.

\begin{lstlisting}[numbers=none]
    // SPDX-License-Identifier: MIT
pragma solidity ^0.8.28;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/utils/ReentrancyGuard.sol";
import "@openzeppelin/contracts/utils/Pausable.sol";
import "@openzeppelin/contracts/utils/Counters.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721URIStorage.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Burnable.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Pausable.sol";

contract TicketMarketplace is
    ERC721,
    ERC721URIStorage,
    ERC721Burnable,
    ERC721Pausable,
    Ownable,
    ReentrancyGuard
{
    uint256 private _tokenIdCounter;

    // Struct untuk menyimpan informasi event
    struct Event {
        uint256 eventId;
        string name;
        string description;
        uint256 price;
        uint256 maxSupply;
        uint256 currentSupply;
        uint256 eventDate;
        bool isActive;
        address organizer;
    }

    // Struct untuk menyimpan informasi tiket
    struct Ticket {
        uint256 tokenId;
        uint256 eventId;
        address owner;
        bool isUsed;
        uint256 purchaseDate;
        uint256 usageDate;
    }

    // Mappings
    mapping(uint256 => Event) public events;
    mapping(uint256 => Ticket) public tickets;
    mapping(uint256 => bool) public eventExists;
    mapping(address => uint256[]) public userTickets;
    mapping(uint256 => address) public authorizedVerifiers;
    mapping(uint256 => bool) public tokenExists; // Track token existence

    // Counters
    uint256 public eventCounter;
    uint256 public totalRevenue;

    // Fee untuk platform (dalam basis points, 250 = 2.5%)
    uint256 public platformFee = 250;
    uint256 public constant MAX_PLATFORM_FEE = 1000; // 10% maksimal

    // Events
    event EventCreated(
        uint256 indexed eventId,
        string name,
        uint256 price,
        uint256 maxSupply
    );
    event TicketPurchased(
        uint256 indexed tokenId,
        uint256 indexed eventId,
        address buyer,
        uint256 price
    );
    event TicketUsed(
        uint256 indexed tokenId,
        uint256 indexed eventId,
        address verifier
    );
    event TicketBurned(uint256 indexed tokenId, uint256 indexed eventId);
    event EventUpdated(uint256 indexed eventId, string name, uint256 price);
    event VerifierAdded(uint256 indexed eventId, address verifier);
    event VerifierRemoved(uint256 indexed eventId, address verifier);

    constructor() ERC721("EventTicket", "ETKT") Ownable(msg.sender) {
        _tokenIdCounter = 0;
        eventCounter = 0;
        totalRevenue = 0;
        platformFee = 250;
    }
    // Modifier untuk memastikan hanya organizer event yang bisa mengakses
    modifier onlyEventOrganizer(uint256 eventId) {
        require(events[eventId].organizer == msg.sender, "Not event organizer");
        _;
    }

    // Modifier untuk memastikan event ada
    modifier eventMustExist(uint256 eventId) {
        require(eventExists[eventId], "Event does not exist");
        _;
    }

    // Modifier untuk verifier yang diotorisasi
    modifier onlyAuthorizedVerifier(uint256 eventId) {
        require(
            authorizedVerifiers[eventId] == msg.sender ||
                events[eventId].organizer == msg.sender ||
                owner() == msg.sender,
            "Not authorized verifier"
        );
        _;
    }

    /**
     * @dev Membuat event baru
     */
    function createEvent(
        string memory name,
        string memory description,
        uint256 price,
        uint256 maxSupply,
        uint256 eventDate
    ) external returns (uint256) {
        require(bytes(name).length > 0, "Event name cannot be empty");
        require(maxSupply > 0, "Max supply must be greater than 0");
        require(
            eventDate > block.timestamp,
            "Event date must be in the future"
        );

        eventCounter++;
        uint256 eventId = eventCounter;

        events[eventId] = Event({
            eventId: eventId,
            name: name,
            description: description,
            price: price,
            maxSupply: maxSupply,
            currentSupply: 0,
            eventDate: eventDate,
            isActive: true,
            organizer: msg.sender
        });

        eventExists[eventId] = true;

        emit EventCreated(eventId, name, price, maxSupply);
        return eventId;
    }

    /**
     * @dev Membeli tiket untuk event tertentu
     */
    function purchaseTicket(
        uint256 eventId,
        string memory newTokenURI
    ) external payable nonReentrant whenNotPaused eventMustExist(eventId) {
        Event storage eventInfo = events[eventId];

        require(eventInfo.isActive, "Event is not active");
        require(
            eventInfo.currentSupply < eventInfo.maxSupply,
            "Event sold out"
        );
        require(msg.value >= eventInfo.price, "Insufficient payment");
        require(
            block.timestamp < eventInfo.eventDate,
            "Event has already passed"
        );

        // Mint NFT ticket
        _tokenIdCounter++;
        uint256 tokenId = _tokenIdCounter;

        // FIXED: Mint ke msg.sender (pembeli), bukan ke organizer
        _safeMint(msg.sender, tokenId);
        _setTokenURI(tokenId, newTokenURI);

        // Mark token as existing
        tokenExists[tokenId] = true;

        // Update event supply
        eventInfo.currentSupply++;

        // FIXED: Create ticket record dengan owner yang benar (msg.sender)
        tickets[tokenId] = Ticket({
            tokenId: tokenId,
            eventId: eventId,
            owner: msg.sender, // FIXED: Pastikan owner adalah pembeli
            isUsed: false,
            purchaseDate: block.timestamp,
            usageDate: 0
        });

        // FIXED: Add to user tickets untuk pembeli, bukan organizer
        userTickets[msg.sender].push(tokenId);

        // Calculate fees
        uint256 platformFeeAmount = (msg.value * platformFee) / 10000;
        uint256 organizerAmount = msg.value - platformFeeAmount;

        // Transfer payment to organizer
        (bool success, ) = payable(eventInfo.organizer).call{
            value: organizerAmount
        }("");
        require(success, "Payment to organizer failed");

        // Keep platform fee in contract
        totalRevenue += platformFeeAmount;

        // Refund excess payment
        if (msg.value > eventInfo.price) {
            uint256 refund = msg.value - eventInfo.price;
            (bool refundSuccess, ) = payable(msg.sender).call{value: refund}(
                ""
            );
            require(refundSuccess, "Refund failed");
        }

        emit TicketPurchased(tokenId, eventId, msg.sender, eventInfo.price);
    }

    /**
     * @dev Menggunakan tiket (verifikasi di pintu masuk)
     */
    function useTicket(
        uint256 tokenId
    ) external onlyAuthorizedVerifier(tickets[tokenId].eventId) {
        require(tokenExists[tokenId], "Ticket does not exist");

        Ticket storage ticket = tickets[tokenId];
        require(!ticket.isUsed, "Ticket already used");

        Event storage eventInfo = events[ticket.eventId];
        require(
            block.timestamp >= eventInfo.eventDate - 1 hours &&
                block.timestamp <= eventInfo.eventDate + 6 hours,
            "Event access window closed"
        );

        // Mark ticket as used
        ticket.isUsed = true;
        ticket.usageDate = block.timestamp;

        emit TicketUsed(tokenId, ticket.eventId, msg.sender);

        // Auto-burn ticket after use
        _burnTicket(tokenId);
    }

    /**
     * @dev Internal function untuk membakar tiket
     */
    function _burnTicket(uint256 tokenId) internal {
        require(tokenExists[tokenId], "Ticket does not exist");

        Ticket storage ticket = tickets[tokenId];

        // FIXED: Remove from user tickets array - gunakan ownerOf untuk mendapatkan owner saat ini
        address ticketOwner = ownerOf(tokenId);
        uint256[] storage userTicketList = userTickets[ticketOwner];
        for (uint256 i = 0; i < userTicketList.length; i++) {
            if (userTicketList[i] == tokenId) {
                userTicketList[i] = userTicketList[userTicketList.length - 1];
                userTicketList.pop();
                break;
            }
        }

        // Mark token as non-existent
        tokenExists[tokenId] = false;

        // Burn the NFT
        _burn(tokenId);

        emit TicketBurned(tokenId, ticket.eventId);
    }

    /**
     * @dev Menambahkan verifier untuk event
     */
    function addVerifier(
        uint256 eventId,
        address verifier
    ) external onlyEventOrganizer(eventId) {
        require(verifier != address(0), "Invalid verifier address");
        authorizedVerifiers[eventId] = verifier;
        emit VerifierAdded(eventId, verifier);
    }

    /**
     * @dev Menghapus verifier untuk event
     */
    function removeVerifier(
        uint256 eventId
    ) external onlyEventOrganizer(eventId) {
        address verifier = authorizedVerifiers[eventId];
        delete authorizedVerifiers[eventId];
        emit VerifierRemoved(eventId, verifier);
    }

    /**
     * @dev Update informasi event
     */
    function updateEvent(
        uint256 eventId,
        string memory name,
        uint256 price
    ) external onlyEventOrganizer(eventId) {
        Event storage eventInfo = events[eventId];
        eventInfo.name = name;
        eventInfo.price = price;

        emit EventUpdated(eventId, name, price);
    }

    /**
     * @dev Mengaktifkan/menonaktifkan event
     */
    function toggleEventStatus(
        uint256 eventId
    ) external onlyEventOrganizer(eventId) {
        events[eventId].isActive = !events[eventId].isActive;
    }

    /**
     * @dev Mendapatkan daftar tiket user
     */
    function getUserTickets(
        address user
    ) external view returns (uint256[] memory) {
        return userTickets[user];
    }

    /**
     * @dev Mendapatkan informasi tiket
     */
    function getTicketInfo(
        uint256 tokenId
    ) external view returns (Ticket memory) {
        require(tokenExists[tokenId], "Ticket does not exist");
        return tickets[tokenId];
    }

    /**
     * @dev Mendapatkan informasi event
     */
    function getEventInfo(
        uint256 eventId
    ) external view returns (Event memory) {
        require(eventExists[eventId], "Event does not exist");
        return events[eventId];
    }

    /**
     * @dev Set platform fee (hanya owner)
     */
    function setPlatformFee(uint256 newFee) external onlyOwner {
        require(newFee <= MAX_PLATFORM_FEE, "Fee too high");
        platformFee = newFee;
    }

    /**
     * @dev Withdraw platform revenue (hanya owner)
     */
    function withdrawRevenue() external onlyOwner {
        uint256 amount = totalRevenue;
        totalRevenue = 0;

        (bool success, ) = payable(owner()).call{value: amount}("");
        require(success, "Withdrawal failed");
    }

    /**
     * @dev Emergency pause contract
     */
    function pause() external onlyOwner {
        _pause();
    }

    /**
     * @dev Unpause contract
     */
    function unpause() external onlyOwner {
        _unpause();
    }

    // Override functions required for multiple inheritance
    function _update(
        address to,
        uint256 tokenId,
        address auth
    ) internal override(ERC721, ERC721Pausable) returns (address) {
        address from = _ownerOf(tokenId);

        // Jika ini bukan mint (from != address(0)) dan bukan burn (to != address(0))
        if (from != address(0) && to != address(0) && tokenExists[tokenId]) {
            // Update ticket owner
            tickets[tokenId].owner = to;

            // Remove dari array userTickets pemilik lama
            uint256[] storage fromUserTickets = userTickets[from];
            for (uint256 i = 0; i < fromUserTickets.length; i++) {
                if (fromUserTickets[i] == tokenId) {
                    fromUserTickets[i] = fromUserTickets[
                        fromUserTickets.length - 1
                    ];
                    fromUserTickets.pop();
                    break;
                }
            }

            // Add ke array userTickets pemilik baru
            userTickets[to].push(tokenId);
        }

        return super._update(to, tokenId, auth);
    }

    function supportsInterface(
        bytes4 interfaceId
    ) public view override(ERC721, ERC721URIStorage) returns (bool) {
        return super.supportsInterface(interfaceId);
    }

    function tokenURI(
        uint256 tokenId
    ) public view override(ERC721, ERC721URIStorage) returns (string memory) {
        return super.tokenURI(tokenId);
    }
}

perintah seperti \emph{npx hardhat run scripts/deploy.ts --network cardona}.

untuk digunakan di integrasi frontend. Setelah perilisan, pengujian dilakukan
untuk memastikan semua fungsi berjalan sesuai spesifikasi, termasuk minting
tiket, transfer tiket, dan verifikasi kepemilikan.

                            if (Math.random() > 0.8) {
                                const tx = await contract.connect(user).purchaseTicket(
                                    eventId,
                                    `https://api.tickets.com/metadata/${Date.now()}`,
                                    { value: ethers.parseEther('0.01') }
                                );
                                const receipt = await tx.wait();

                                await metricsCollector.record(
                                    'purchaseTicket',
                                    networkName,
                                    receipt.gasUsed.toString(),
                                    Date.now() - opStartTime,
                                    receipt.status === 1,
                                    null,
                                    receipt
                                );
                            } else {
                                await contract.getEventInfo(eventId);

                                await metricsCollector.record(
                                    'getEventInfo',
                                    networkName,
                                    '0',
                                    Date.now() - opStartTime,
                                    true
                                );
                            }
                        } catch (error) {
                            await metricsCollector.record(
                                'mixed_operation',
                                networkName,
                                '0',
                                Date.now() - opStartTime,
                                false,
                                error
                            );
                        }
                    })();

                    batchPromises.push(promise);
                }

                await Promise.allSettled(batchPromises);

                const elapsed = Date.now() - startTime;
                const progress = ((elapsed / testDuration) * 100).toFixed(1);
                if (elapsed % 30000 < batchInterval) {
                    console.log(`Progress: ${progress}% (${Math.round(elapsed / 1000)}s)`);
                }

                await new Promise(resolve => setTimeout(resolve, batchInterval));
            }

            console.log('Sustained load test completed');
        });
    });

    after(async function () {
        const report = metricsCollector.exportToFile('load-test-results.json');

        console.log('\n=== OPTIMIZED LOAD TEST SUMMARY ===');

        Object.keys(report.networkComparison).forEach(network => {
            const stats = report.networkComparison[network];
            console.log(`\n${network.toUpperCase()} Network Results:`);
            console.log(`Total Transactions: ${stats.totalTransactions}`);
            console.log(`Success Rate: ${stats.successRate}%`);
            console.log(`Average Gas Used: ${stats.avgGasPerTx}`);
            console.log(`Average Response Time: ${stats.avgResponseTime}ms`);
            console.log(`Estimated TPS: ${stats.estimatedTPS}`);

            console.log('Operations Breakdown:');
            Object.keys(stats.operations).forEach(op => {
                const opStats = stats.operations[op];
                const opSuccessRate = ((opStats.successCount / opStats.count) * 100).toFixed(2);
                const avgGas = opStats.count ? Math.round(opStats.totalGas / opStats.count) : 0;
                console.log(`  ${op}: ${opStats.count} ops, ${opSuccessRate}% success, avg gas: ${avgGas}`);
            });
        });
    });
});
